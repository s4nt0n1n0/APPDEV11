# Project Documentation: Personal Portfolio

## 1. Objective
This project is a personal professional portfolio for an IT student, showcasing development skills, API integrations, and transaction-based features.

## 2. API Integrations (At least 3 required)
The following professional APIs have been integrated into this project:

- **GitHub API**: Used on the `IT Journey` page to dynamically fetch and display the student's latest repositories. This ensures the portfolio is always up-to-date with live coding projects.
- **EmailJS API**: Integrated into the `Contact` page to handle professional email delivery. This allows users to send messages directly from the website without needing a backend server.
- **Leaflet Map API**: An interactive map integration using OpenStreetMap data to display the student's professional location in Daet, Camarines Norte.

## 3. Transaction Features (At least 2 required)
The following user-action transactions are implemented with JavaScript validation and visible feedback:

- **Contact Form Submission**: Users can submit their Name, Email, and Message. The transaction includes field validation and a visible success/error response after processing through the EmailJS API.
- **Feedback/Testimonial Submission**: A simulated database transaction on the `IT Journey` page. Users can leave testimonials which are "recorded" and immediately displayed in the "Latest Feedback" section, demonstrating data processing and dynamic content rendering.

## 4. Design & Usability
The website follows a "Cute Vibe" aesthetic:
- **Responsive Design**: Fully functional on mobile, tablet, and desktop.
- **Navigation**: Working links across all sections: Home, About Me, Skills and Projects, and Contact.
- **Consistent UI**: Unified color palette (Pink & Pearl), rounded corners (25px), and smooth hover animations.
